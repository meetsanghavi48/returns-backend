💡 How rules work
• Rules run top-down by priority
• On Submit: fires when customer submits request
• On Carrier Event: fires when Delhivery updates AWB
• On Approval: fires when you approve a request
• On Inspection: fires when item marked as inspected
• Match ALL: every condition must be true
• Match ANY: at least one condition must be true
📋 Available conditions
Request type — return / exchange / mixed
Return reason — text the customer selected
Request stage — pending / approved / received…
Carrier event — pickup / in_transit / delivered
Order value — total order amount (₹)
Product price — price of returned item (₹)
Payment method — COD or Prepaid
Refund method — what customer chose
Return count — how many times customer returned
Days since order — age of the order
